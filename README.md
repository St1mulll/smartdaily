# SmartDaily — деплой на Netlify

## Що в цьому пакеті
```
index.html              — головна сторінка сайту
netlify.toml             — налаштування кешування і заголовків безпеки
_redirects               — фоллбек на index.html для будь-яких шляхів (захист від 404)
robots.txt               — дозвіл індексації пошуковикам + посилання на sitemap
sitemap.xml              — мінімальна карта сайту (1 сторінка)
manifest.json             — PWA-маніфест (для "Додати на головний екран" в Android)
favicon.ico               — класична іконка для табу браузера (16/32/48px мульти-розмір)
favicon-16x16.png          — іконка для табу браузера
favicon-32x32.png          — іконка для табу браузера (Retina)
favicon-512.png            — велика іконка для PWA/Android
apple-touch-icon.png       — іконка для iOS (180×180px)
```

## Спосіб 1 — Drag & Drop (найшвидше, без Git)

1. Зайди на **app.netlify.com** → залогінься (або створи акаунт безкоштовно)
2. На дашборді знайди область **"Want to deploy a new site without connecting to Git?"**
3. Перетягни **всю папку** (або всі 5 файлів одночасно) у цю область
4. Netlify за 10-30 секунд задеплоїть і видасть лінк типу `https://random-name-123.netlify.app`
5. Перевір що сайт відкривається і виглядає коректно

## Спосіб 2 — Через Git (якщо плануєш далі редагувати сайт)

1. Створи репозиторій на GitHub, заклади туди ці файли
2. На Netlify: **Add new site → Import an existing project → GitHub**
3. Обери репозиторій, **Build command залиш порожнім**, **Publish directory: `.`**
4. Deploy

## Після деплою — обов'язково зроби це

### 1. Підключи свій домен (якщо є)
Netlify → твій сайт → **Domain settings → Add custom domain**
Якщо домену ще немає — можеш користуватись безкоштовним `*.netlify.app` для тестів, але для реклами в Facebook/Instagram краще купити нормальний домен (напр. на reg.ua, namecheap).

### 2. ⚠️ ВАЖЛИВО: заміни домен-заглушку
Я використав **тимчасовий домен `smartdaily.com.ua`** як приклад у трьох місцях:
- `index.html` → `<meta property="og:url" content="...">`
- `robots.txt` → `Sitemap: https://smartdaily.com.ua/sitemap.xml`
- `sitemap.xml` → `<loc>https://smartdaily.com.ua/</loc>`

Якщо твій реальний домен інший (або ти ще не купив домен і поки використовуєш `*.netlify.app`) —
заміни ці 3 посилання на справжню адресу сайту після деплою.

### 3. Favicon — вже готовий ✅
Я згенерував фірмову іконку SmartDaily (оранжевий душ із краплями) у всіх потрібних розмірах:
`favicon.ico`, `favicon-16x16.png`, `favicon-32x32.png`, `favicon-512.png`, `apple-touch-icon.png`, `manifest.json`.
Вони вже лежать в цій папці поруч з `index.html` і підключені в коді — нічого додатково робити не треба,
просто задеплой всі файли разом.

### 4. og:image — заміни на справжній банер
Зараз у `og:image` стоїть фото товару (премium-варіант) як тимчасове рішення.
Якщо хочеш красиве прев'ю при розшарюванні лінка в Telegram/Instagram —
зроби банер **1200×630px** з лого + назвою + фото товару, заливай на Cloudinary
і встав посилання замість поточного в `<meta property="og:image" content="...">`
та `<meta name="twitter:image" content="...">`.

### 5. Перевірка після деплою
- **PageSpeed**: pagespeed.web.dev → встав URL → перевір Performance/Accessibility/Best Practices/SEO
- **OG-прев'ю**: opengraph.xyz → встав URL → перевір як виглядає картка лінка
- Перевір що форма замовлення реально шле повідомлення в Telegram (напиши тестове замовлення)

## ⚠️ Безпека Telegram-бота (нагадування)
BOT_TOKEN і CHAT_ID зараз прописані прямо в коді сторінки (видно у "Просмотр кода" будь-кому).
Це працює, але теоретично хтось може вкрасти токен. Для production варто перенести
відправку повідомлень через серверну функцію (Netlify Functions — це безкоштовно і просто),
але для старту з невеликим трафіком це не блокер.
